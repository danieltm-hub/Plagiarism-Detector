# /scaffold [template]

Generate project structure using [template].

## Workflow

### Phase 1: Template Selection
Available templates:
- `python-package` — Library with pyproject.toml
- `python-service` — FastAPI/Flask service
- `typescript-package` — NPM library
- `typescript-service` — Express/NestJS service
- `documentation` — MkDocs site
- `minimal` — Basic structure only

### Phase 2: Configuration
Gather from user:
- Project name
- Description
- Author
- License
- Key dependencies

### Phase 3: Generation
Create structure:
```
{project-name}/
├── [template-specific files]
├── makefile
├── README.md
└── .knowledge/
    └── plans/
        └── initial-setup.md
```

### Phase 4: Initial Plan
Create `.knowledge/plans/active/initial-setup.md`:
```yaml
---
id: initial-setup
created: {date}
type: plan
status: active
phases:
  - name: Validate structure
    done: false
  - name: Configure dependencies
    done: false
  - name: Initial commit
    done: false
---

# Initial Setup Plan

Generated by `/scaffold {template}`

## Next Steps
1. Review generated files
2. Customize configuration
3. Install dependencies
4. Make initial commit
```

## Key Constraints
- Idempotent — running twice is safe
- Opinionated — uses framework best practices
- Documented — always creates initial plan
